🚀Changelog

Tracking every iteration of Ling Player

0.11.0.0 2026-06-12 Alpha
✨ New Features
  • Taskbar Playback Control: Hover your mouse over the taskbar icon to access previous track, next track, play, and pause controls — making playback control more convenient.
  • Lyrics Editing: You can now modify and save embedded lyrics for tracks in the metadata editing dialog.
0.10.0.0 2026-06-09 Alpha
✨ New Features
  • UI Scale Option: You can now adjust the size of interface elements. Path: Settings -> UI
  • Navigation Bar Expanded Width Setting: You can now adjust the expanded width of the left navigation bar within a range of [100,480]. Path: Settings -> UI
  • New Language Support: Added support for Traditional Chinese, English(UK), Japanese, Korean, Russian, German, French, Portuguese (Brazil), Spanish, Polish, and Turkish. Translations for these languages may contain errors; feedback is welcome.
⚡ Improvements
  • Lyric Line Switching: Added fade-in/fade-out effect.
  • Lyrics Font Size and Weight Adjustment: Normal and highlighted lyric text now use the same font size and weight.
🔧 Bug Fixes
  • Fixed an issue where artists could not be dragged in the album edit dialog.
  • Fixed a potential crash when right-clicking on the album page.
  • Fixed an issue where the close button became unresponsive after closing the window and reopening it via the taskbar button.
  • Fixed an issue where the default icon was not shown when a track had no cover.
  • Fixed an issue where the end of a track was being truncated.
  • Fixed an issue where auto-scrolling after clicking a track in the playback queue caused the currently playing track to become invisible.
  • Fixed an issue where the previous track's cover and lyrics were still displayed when a track file failed to load.
  • Fixed an issue where existing lyrics were lost when saving track metadata.
0.9.0.0 2026-06-06 Alpha
✨ New Features
  • Track Metadata Editing: Supports modifying track cover, title, artist, and year. Changes to track metadata will be written to the track file by default. Note that editing metadata for tracks from CUE files is currently not supported. Path: Right-click a track -> Select Edit Metadata
  • Album Metadata Editing: Supports modifying album title, artist, and year. You can choose to apply album title, artist, year, and cover to all tracks in the album (will be written to corresponding track files). Note that editing album metadata is not supported when the album contains tracks from CUE files. Path: Right-click an album -> Select Edit Metadata
  • Full Scan Option: When enabled, rescans all tracks to ensure track data is up to date. Path: Settings -> Library -> Full scan
⚡ Improvements
  • Included Folders: When adding a new folder, if any subfolders of that folder already exist in the list, those subfolders are automatically removed.
  • CUE File Parsing: Removes possible double quotes attached to the genre field.
  • Page Navigation: After clicking a folder on the folder page, the page indicator stays on the folder menu instead of switching to the track menu; similar adjustments have been made for the playlist page.
  • Thumbnail Generation: Generates thumbnails with a uniform height of 200 pixels. No longer generates different thumbnail sizes for tracks, albums, and artists, reducing disk space usage.
  • Artist/Genre Parsing: Intelligently recognizes and preserves manually modified track artist, album artist, and genre information, preventing them from being overwritten by original tags.
🔧 Bug Fixes
  • Fixed an issue where the Add Playlist dialog was too small.
  • Fixed a potential crash when right-clicking on the album page.
  • Fixed an issue where clicking the "Library" button on the track page with no tracks present would incorrectly select the "Settings" menu after navigating.
  • Fixed an issue where saving after deleting an artist while editing album metadata would incorrectly delete that artist.
  • Fixed an issue where rescanning failed to update track information after modifying track metadata externally.
  • Fixed an issue where in some cases, audio files associated with CUE files were incorrectly added to the library as regular track files.
  • Fixed an issue where edited albums or artists could not be removed even when they met the removal conditions.
  • Fixed an issue where incorrect scan results were displayed after dragging folders or multiple track files onto the track or playlist page.
  • Fixed an issue where view settings were not correctly applied when switching to the album page for the first time.
0.8.1.0 2026-05-24 Alpha
⚡ Improvements
  • Mini Window Playlist Operations: Added a playlist operation page for the mini window. Click the playlist button in the bottom-right corner to open it. Supports switching tracks, removing tracks from the queue, clearing the queue, and more.
  • Mini Window Settings: Added a settings page for the mini window. Click the settings button on the far right to open it. Currently includes the following settings: theme (synced with global settings), overlay opacity, and whether to show the mini window icon in the taskbar.
  • Removed Corner Radius from Mini Window Track Cover: Removed the corner radius effect from the track cover in the mini window to avoid black or white areas appearing at the window corners.
  • Default Value Adjustment for Duration Filter: The default value for the duration filter setting has been adjusted to 0 to avoid affecting scan results when scanning tracks for the first time.
🔧 Bug Fixes
  • Fixed an issue where content overflowed on the album details page when the description text was too long.
  • Fixed an issue where content overflowed and the page could not scroll on the artist details page when the description text was too long.
  • Fixed an issue where an image could not be set for an artist when the artist name contained special characters.
0.8.0.0 2026-05-22 Alpha
✨ New Features
  • Rendering Mode Option: When experiencing rendering issues such as flickering or display problems, you can try switching the rendering mode. Path: Settings -> UI -> Rendering mode.
  • Automatic Log File Cleanup: The application will automatically delete log files older than 7 days on startup. Additionally, you can click Clear Cache under Log Cache on the About page to delete all log files at any time.
🔧 Bug Fixes
  • Fixed an issue where the application failed to start when the settings file was invalid.

0.7.0.0 2026-05-17 Alpha

0.5.4.0 2026-05-12 Alpha
⚡ Improvements
  • Album Merging Rule: Before merging albums, the player will check for conflicts in disc numbers or track numbers. If conflicts exist, they will be automatically split into separate albums.
  • Default Sorting for Albums/Artists: The default sorting rule has been adjusted to ascending order by name.
  • Added "Show Contributing Artists" Switch: Control whether to show featured artists on the artist page. Disabled by default. You can find the ""Show featured artists on artist page"" option in Settings -> UI to enable it.
  • Auto Cleanup After Scanning: After scanning completes, albums without any tracks and artists without any albums or tracks will be automatically deleted. If an album has been edited (e.g., year, cover, album artist), it will be retained; the same applies to artists.
  • Navigation Menu Settings Adjustment: Navigation menu settings have been moved to a standalone settings page. Path: Settings -> Navigation Menu
